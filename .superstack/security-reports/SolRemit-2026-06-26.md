## Findings

### [MEDIUM] SR-001: Transitive `ws` DoS advisory via CDP SDK dependency tree
- **Confidence:** 9/10 · **Phase 3 / OWASP A03 (Supply Chain)**
- **Location:** `node_modules/@coinbase/cdp-sdk → viem → ws` (GHSA-96hv-2xvq-fx4p); 10 advisories total, all transitive via `@coinbase/cdp-{core,hooks,sdk}`.
- **Description:** `npm audit` reports a high-severity `ws` memory-exhaustion DoS plus moderates, all pulled in by the CDP SDK chain (viem/ws; react-native/expo). Our **direct** `ws` is 8.21.0 (not vulnerable). Real exploitability here is limited: the `ws` advisory targets a ws *server* receiving crafted frames, whereas viem uses `ws` as a *client*; and the Solana-only flows may not touch viem's hot path. Still material for a money app's dependency posture.
- **Remediation:**
  - Do **NOT** run `npm audit fix --force` — it downgrades `next`→9.3.3 and `cdp-core`→0.0.0 (breaks the app).
  - Pin a safe `ws` via `package.json` `overrides`: `"overrides": { "ws": "^8.18.3" }` (resolves nested copies to ≥8.21.0), then re-audit.
  - Track CDP SDK releases; update when they bump viem/ws.
- **Priority:** P2

### [MEDIUM] SR-002: No rate limiting on cost/money-bearing API routes
- **Confidence:** 8/10 · **Phase 9 / OWASP A06 (Insecure Design)**
- **Location:** `app/api/fx/{quote,order,execute}/route.ts`, `app/api/onramp/{buy-options,buy-quote}/route.ts`
- **Description:** None of the routes enforce per-IP/per-user rate limiting. `/api/onramp/buy-quote` consumes CDP API quota and `/api/fx/*` consumes Jupiter quota; unbounded calls enable cost-amplification and upstream rate-limit exhaustion (DoS for legitimate users). Jupiter's own 429s partially backstop FX routes, but not the onramp ones.
- **Exploit scenario:** An attacker scripts thousands of `POST /api/onramp/buy-quote` calls, burning CDP quota / racking cost and tripping upstream limits so real users can't get quotes.
- **Remediation:** Add edge/middleware rate limiting (e.g. Upstash Ratelimit or Vercel WAF) keyed by IP (and by authenticated user once sign-in is required), tightest on `execute` and `onramp/*`.
- **Priority:** P2

## INFO (below daily gate — defense-in-depth)
- **Upstream error text returned to client.** `/api/onramp/*` returns `await res.text()` and other routes return `err.message`. Low info-disclosure risk; prefer a generic client message + server-side log. (~5/10)
- **CDP outbound fetches lack an explicit timeout.** `jupiterFetch` has a 5s AbortController timeout; the CDP onramp `fetch()` calls don't — a hung upstream could stall the route. Add a timeout. (~6/10)

## Verified GOOD (no action)
- **Secrets hygiene:** no secrets committed (git history clean); `.env`/`.env.local` gitignored; `.env.example` holds placeholders only.
- **Env separation correct:** secrets (`CDP_API_KEY_*`, `JUPITER_API_KEY`) are server-only (no `NEXT_PUBLIC_`); only the non-secret `NEXT_PUBLIC_CDP_PROJECT_ID` is public.
- **No secret logging** (no `console.*` in app code).
- **Signing safety:** private keys never touch our server — signing is client-side via CDP embedded wallet (`useSignSolanaTransaction`); we only relay a signed blob to Jupiter.
- **Fail-closed:** `getCDPCredentials()` throws if unset; `getMidMarketRate()` throws on missing/low-confidence price (no fabricated rate shown).
- **Input validation:** `usd` bounded (0–50000), `taker` length-checked, onramp required-fields checked; inputs go through `URLSearchParams` (no injection).
- **No SSRF:** all outbound hosts are hardcoded (Jupiter, CDP); no user-controlled URLs.
- **No dangerous sinks:** no `eval`/`new Function`/`innerHTML`/`dangerouslySetInnerHTML`/`child_process`/TLS-bypass.
- **Untrusted-token defense:** destination stablecoin is trust-checked (`assessTrust`) and cross-checked against the expected mint before routing.

## Remediation applied (2026-06-26)
- **SR-001 (partially resolved):** Added `overrides: { "ws": "$ws" }` — the vulnerable nested
  `ws` (under viem) is now deduped away. Production audit: **10 vulns (5 high) → 7 moderate, 0 high**.
  Remaining 7 are transitive moderates (Next-bundled `postcss` XSS-on-stringify; `uuid` bounds-check
  via `@solana/web3.js`→CDP) with low real exploitability and upstream-dependent fixes — NOT
  force-pinned (would risk breaking Next). Tracked for CDP/Next updates.
- **SR-002 (resolved):** Added `lib/rate-limit.ts` (in-memory fixed-window) to all five money/cost
  routes: fx-quote 60/min, fx-order 20/min, fx-execute 10/min, onramp-options 20/min,
  onramp-quote 20/min. Verified live: route returns 429 after the limit. Note: in-memory is
  per-instance — production should back this with Upstash/WAF for multi-instance hosting.
